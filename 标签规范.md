## 📋 标签使用规范

### 标签命名规则
1. **一致性**：同一概念使用统一标签名
2. **简洁性**：标签名简洁明了，避免过长
3. **中文优先**：优先使用中文标签，便于理解
4. **层次性**：通过标签体现知识层次

### 标签使用建议
1. **每个文档3-8个标签**：避免标签过多或过少
2. **必须包含技术栈标签**：明确技术归属
3. **建议包含重要程度标签**：便于优先级排序
4. **问题文档必须包含问题类型标签**：便于问题查找

### 标签组合示例

#### 技术原理文档
```
tags: [HashMap原理, Java, 核心概念, 重要, 已掌握]
```

#### 问题解决文档
```
tags: [性能优化, MySQL, 查询慢, 问题解决, 实践]
```

#### 实战应用文档
```
tags: [SpringBoot, 自动配置, 实践, 项目必备, 学习中]
```
